背景

复习数据库理论知识时，讲到的约束相关的知识，其实这个知识点，几乎每个DDL语句都有，只是没有注意。

概述

Integrity constraints（完整性约束）是数据库中的规则，用于确保数据的准确性、一致性和完整性。

这些约束通过限制可以插入、更新或删除的数据，防止错误数据进入数据库，从而保证数据库中的数据保持有效和符合业务逻辑。

域约束（Domain Constraint）

这是对列中允许存储的值的限制。域约束确保列中的每个值都符合指定的数据类型、值范围和格式。例如，年龄列只能包含正整数，日期列只能包含有效的日期值。

具体使用就是对每个字段的类型定义，以及使用check语句做范围限制。

比如：

CREATE TABLE employees (
    age INT CHECK (age > 0 AND age < 150)  -- 年龄必须在0到150之间
);

实体完整性约束（Entity Integrity Constraint）

这是通过主键约束（Primary Key Constraint）实现的，确保每个表中的记录都有一个唯一标识。主键约束确保表中的每一行（记录）都是唯一的，且不能为空。

比如：

CREATE TABLE employees (
    employee_id INT PRIMARY KEY,  -- employee_id 必须唯一且非空
    first_name VARCHAR(50)
);

参照完整性约束（Referential Integrity Constraint）

通过外键约束（Foreign Key Constraint）来实现，确保表之间的关系一致性。外键约束确保一个表中的列的值必须存在于另一个表中的主键列中，从而维护表之间的关系。这样可以防止无效的引用，例如在子表中引用一个不存在的父记录。

比如：

CREATE TABLE departments (
    department_id INT PRIMARY KEY,
    department_name VARCHAR(50)
);

CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    department_id INT,
    FOREIGN KEY (department_id) REFERENCES departments(department_id)  -- department_id 必须在 departments 表中存在
);

但是，出于性能和降低复杂性方面的考虑，一般不使用外键约束，而是使用在应用系统上面管理不同表的关联关系。

唯一性约束（Unique Constraint）

唯一性约束确保某个列或列的组合中的值在整个表中是唯一的，不能重复。与主键不同，唯一性约束允许列包含空值（NULL），但每个非空值必须是唯一的。

比如：

CREATE TABLE employees (
    email VARCHAR(100) UNIQUE  -- email 列中的值必须是唯一的
);

唯一约束可以对多个字段一起做唯一性约束。

非空约束（NOT NULL Constraint）

非空约束用于确保列中不能有空值。它强制要求在插入或更新记录时，列必须有值，不能留空。

比如：

CREATE TABLE employees (
    first_name VARCHAR(50) NOT NULL  -- first_name 列不能为空
);

检查约束（Check Constraint）

检查约束用于限制列中的值必须满足某个条件。它可以通过 CHECK 关键字定义，例如确保某个数值在一定的范围内，或者某个日期在特定的范围之内。

比如：

CREATE TABLE employees (
    salary DECIMAL(10, 2),
    CHECK (salary >= 0)  -- salary 必须是非负数
);

总结

Integrity constraints 是数据库设计中非常重要的部分，它们通过约束规则来确保数据的完整性和一致性。通过合理使用各种完整性约束，可以防止不合法的数据进入数据库，从而确保数据库中的数据准确可靠，并且符合预期的业务逻辑。
