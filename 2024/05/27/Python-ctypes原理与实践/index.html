背景

本文主要记录下Python ctypes的基本原理，以及如何通过ctypes调用C语言的动态链接库，如果一个库没有Python的版本，只有C/C++版本，那么就可以通过这种方式调用底层函数，实现同样的功能。

Ctypes简介

ctypes 是 Python 的一个标准库模块，用于与 C 语言编写的动态链接库（DLLs 或 shared libraries）进行交互。使用 ctypes 可以在 Python 中调用 C 函数，并通过这种方式扩展 Python 的功能和性能。以下是使用 ctypes 的一些主要好处：

  1. 性能提升：C 语言的执行速度通常比纯 Python 代码快很多，特别是在涉及大量数值计算或需要高效内存管理的情况下。通过 ctypes 调用 C 代码，可以显著提高这些性能瓶颈部分的执行速度。
  2. 重用现有 C 代码：很多高效的算法和库都是用 C 语言编写的。通过 ctypes，你可以直接在 Python 中重用这些现有的 C 代码库，而不必重新实现。
  3. 访问底层系统接口：有时候，需要访问底层系统功能或者硬件接口，而这些功能可能在高级语言（如 Python）中并不直接可用。使用 ctypes 可以调用操作系统的底层 API，从而实现这些高级语言难以完成的功能。比如获取线程被哪个逻辑核调度的功能。
  4. 内存管理的灵活性：C 语言提供了更多的内存管理控制，通过 ctypes 可以在 Python 中利用这些控制，比如管理内存分配和释放、操作内存地址等。这对于需要高度优化的应用程序（如游戏开发或实时系统）非常重要。
  5. 跨平台兼容性：ctypes 是一个跨平台的库，支持在各种操作系统（如 Windows、Linux、macOS）上使用。这意味着你可以编写一次 C 代码，并在多个平台上通过 ctypes 调用。
  6. 动态加载库：ctypes 允许动态加载共享库，这意味着可以在运行时决定加载哪些库和函数。这对于开发插件系统或模块化应用程序非常有用。
  7. 方便的接口：相比于其他与 C 语言交互的方式（如编写 Python 扩展模块），ctypes 提供了更方便的接口，不需要编写复杂的 C 扩展代码，只需要定义好函数和数据类型即可直接调用。

ctypes简单实例

a + b

下面通过一个简单的实例，了解ctypes如何使用

一个简单的a+b的C代码：

int add(int a, int b) {
    return a + b;
}

编程成ab.so文件，动态链接库：

$ gcc -shared -o ab.so -fPIC ab.c

使用Python调用这个库中的add方法：

import ctypes

# 加载共享库
example = ctypes.CDLL('./ab.so')

# 定义函数原型 参数和返回值
example.add.argtypes = (ctypes.c_int, ctypes.c_int)
example.add.restype = ctypes.c_int

# 调用函数
result = example.add(3, 5)
print(f"Result: {result}")  # 输出：Result: 8

运行结果：

$ python3 add.py
Result: 8

这是一个自己编写的C代码，如何通过编译成动态链接库的方式调用的。

pthread_self

C语言里面有这么一个和线程有关的函数，但是Python没有一模一样的，这里使用pthread的库的函数，获取主线程的id。

import ctypes

# 加载 pthread 库
thread_lib = ctypes.CDLL('libpthread.so.0')

# 打印加载的库对象
print(thread_lib)

# 正确设置 pthread_self 的返回类型为 ctypes.c_long
thread_lib.pthread_self.restype = ctypes.c_long

# 调用 pthread_self 函数并获取线程 ID
thread_id = thread_lib.pthread_self()

# 打印主线程 ID
print('main thread id is', thread_id)

运行结果：

它提供了一组用于捕获和分析网络数据包的函数，支持多种操作系统，包括 Unix-like 系统（如 Linux 和 macOS）和 Windows（通过 WinPcap 或 Npcap）。

我们通过这个方法的调用，学习一下较为复杂的C函数是如何调用的：

import ctypes
from ctypes import POINTER, c_char_p, c_int, c_uint, c_void_p

# 定义 pcap_if_t 结构体
class pcap_if_t(ctypes.Structure):
    pass

pcap_if_t._fields_ = [
    ("next", POINTER(pcap_if_t)),
    ("name", c_char_p),
    ("description", c_char_p),
    ("addresses", c_void_p),
    ("flags", c_uint)
]

# 加载库
libpcap = ctypes.CDLL("libpcap.so.1")

# 定义 pcap_findalldevs 函数的参数和返回类型 POINTER(POINTER)是二级指针
libpcap.pcap_findalldevs.argtypes = [POINTER(POINTER(pcap_if_t)), c_char_p]
# 返回结果是一个整数
libpcap.pcap_findalldevs.restype = c_int

# 创建 pcap_if_t 指针和错误缓冲区
alldevs = POINTER(pcap_if_t)()
# 错误占位符
errbuf = ctypes.create_string_buffer(256)

# 调用函数
result = libpcap.pcap_findalldevs(ctypes.byref(alldevs), errbuf)
if result == 0:
    print("Success")
else:
    print("Error:", errbuf.value.decode())

# 遍历并打印设备信息 这是一个单链表，遍历单链表，打印设备信息
cur = alldevs
while cur:
    print(f"Name: {cur.contents.name.decode()}")
    print(f"Description: {cur.contents.description.decode() if cur.contents.desc
ription else 'No description'}")
    print(f"Flags: {cur.contents.flags}")
    print(f"Addresses: {cur.contents.addresses}")
    print()
    cur = cur.contents.next

# 释放设备列表
libpcap.pcap_freealldevs(alldevs)

一个简单算法题的ctypes调用，测试和Python的性能差距

取自 LeetCode 至少是其他数字两倍的最大数

C语言实现如下：

dominant_index.c:

#include <stdbool.h>

int dominantIndex(int* nums, int numsSize) {
    int max_num = nums[0];
    int i;
    int max_index = 0;
    for (i = 0 ; i < numsSize; i++) {
        if (nums[i] >= max_num) {
            max_num = nums[i];
            max_index = i;
        }
    }
    bool all_double = 1;
    for (i = 0; i < numsSize; i++) {
        if (i != max_index && max_num - nums[i] < nums[i]) {
            all_double = 0;
            break;
        }
    }
    return all_double == 0 ? -1 : max_index;
}

编译为so文件：

$ gcc -shared -o libdominantindex.so -fPIC dominant_index.c

调用：

import ctypes
import numpy as np
import time

# Load the shared library
lib = ctypes.CDLL('./libdominantindex.so')

# Define the function argument and return types
lib.dominantIndex.argtypes = (ctypes.POINTER(ctypes.c_int), ctypes.c_int)
lib.dominantIndex.restype = ctypes.c_int

def dominant_index(nums):
    nums_array = np.array(nums, dtype=np.int32)
    nums_ptr = nums_array.ctypes.data_as(ctypes.POINTER(ctypes.c_int))
    result = lib.dominantIndex(nums_ptr, len(nums))
    return result

# Example usage with timing

纯Python实现：

class Solution:
    def dominantIndex(self, nums):
        max_num = max(nums)
        all_double = True
        for i in nums:
            if i != max_num and max_num - i < i:
                all_double = False
                break
        if all_double:
            return nums.index(max_num)
        else:
            return -1

a = Solution()
print(a.dominantIndex(list(range(10000000))))
print(a.dominantIndex(list(range(100000))))

Ctypes原理简单说明

简单列下ctypes的__init__.py的代码：

"""create and manipulate C data types in Python"""

import os as _os, sys as _sys
import types as _types

__version__ = "1.1.0"

from _ctypes import Union, Structure, Array
from _ctypes import _Pointer
from _ctypes import CFuncPtr as _CFuncPtr
from _ctypes import __version__ as _ctypes_version
from _ctypes import RTLD_LOCAL, RTLD_GLOBAL
from _ctypes import ArgumentError

from struct import calcsize as _calcsize


def create_string_buffer(init, size=None):
    """create_string_buffer(aBytes) -> character array
    create_string_buffer(anInteger) -> character array
    create_string_buffer(aBytes, anInteger) -> character array
    """
    if isinstance(init, bytes):
        if size is None:
            size = len(init)+1
        _sys.audit("ctypes.create_string_buffer", init, size)
        buftype = c_char * size
        buf = buftype()
        buf.value = init
        return buf
    elif isinstance(init, int):
        _sys.audit("ctypes.create_string_buffer", None, init)
        buftype = c_char * init
        buf = buftype()
        return buf
    raise TypeError(init)
    
    
class CDLL(object):
    """An instance of this class represents a loaded dll/shared
    library, exporting functions using the standard C calling
    convention (named 'cdecl' on Windows).

    The exported functions can be accessed as attributes, or by
    indexing with the function name.  Examples:

    <obj>.qsort -> callable object
    <obj>['qsort'] -> callable object

    Calling the functions releases the Python GIL during the call and
    reacquires it afterwards.
    """
    _func_flags_ = _FUNCFLAG_CDECL
    # 默认返回值是int的
    _func_restype_ = c_int
    # default values for repr
    _name = '<uninitialized>'
    _handle = 0
    _FuncPtr = None
        # 参数一般就是一个so文件的名称，比如ab.so等。
    def __init__(self, name, mode=DEFAULT_MODE, handle=None,
                 use_errno=False,
                 use_last_error=False,
                 winmode=None):
        self._name = name
        flags = self._func_flags_
        if use_errno:
            flags |= _FUNCFLAG_USE_ERRNO
        if use_last_error:
            flags |= _FUNCFLAG_USE_LASTERROR
        if _sys.platform.startswith("aix"):
            """When the name contains ".a(" and ends with ")",
               e.g., "libFOO.a(libFOO.so)" - this is taken to be an
               archive(member) syntax for dlopen(), and the mode is adjusted.
               Otherwise, name is presented to dlopen() as a file argument.
            """
            if name and name.endswith(")") and ".a(" in name:
                mode |= ( _os.RTLD_MEMBER | _os.RTLD_NOW )
        if _os.name == "nt":
            if winmode is not None:
                mode = winmode
            else:
                import nt
                mode = nt._LOAD_LIBRARY_SEARCH_DEFAULT_DIRS
                if '/' in name or '\\' in name:
                    self._name = nt._getfullpathname(self._name)
                    mode |= nt._LOAD_LIBRARY_SEARCH_DLL_LOAD_DIR

        class _FuncPtr(_CFuncPtr):
            _flags_ = flags
            _restype_ = self._func_restype_
        self._FuncPtr = _FuncPtr
                # 主要是从这里打开了一个handler
        if handle is None:
            self._handle = _dlopen(self._name, mode)
        else:
            self._handle = handle

    def __repr__(self):
        return "<%s '%s', handle %x at %#x>" % \
               (self.__class__.__name__, self._name,
                (self._handle & (_sys.maxsize*2 + 1)),
                id(self) & (_sys.maxsize*2 + 1))

    def __getattr__(self, name):
        # 通过名称获取属性，主要是方法
        if name.startswith('__') and name.endswith('__'):
            raise AttributeError(name)
        func = self.__getitem__(name)
        setattr(self, name, func)
        return func

    def __getitem__(self, name_or_ordinal):
        # FuncPtr是函数的指针
        func = self._FuncPtr((name_or_ordinal, self))
        if not isinstance(name_or_ordinal, int):
            func.__name__ = name_or_ordinal
        return func
      
      
# 这里定义了CFUNCTYPE的方法，返回CFunctionType，而且做了一层缓存。
_c_functype_cache = {}
def CFUNCTYPE(restype, *argtypes, **kw):
    """CFUNCTYPE(restype, *argtypes,
                 use_errno=False, use_last_error=False) -> function prototype.

    restype: the result type
    argtypes: a sequence specifying the argument types

    The function prototype can be called in different ways to create a
    callable object:

    prototype(integer address) -> foreign function
    prototype(callable) -> create and return a C callable function from callable
    prototype(integer index, method name[, paramflags]) -> foreign function calling a COM method
    prototype((ordinal number, dll object)[, paramflags]) -> foreign function exported by ordinal
    prototype((function name, dll object)[, paramflags]) -> foreign function exported by name
    """
    flags = _FUNCFLAG_CDECL
    if kw.pop("use_errno", False):
        flags |= _FUNCFLAG_USE_ERRNO
    if kw.pop("use_last_error", False):
        flags |= _FUNCFLAG_USE_LASTERROR
    if kw:
        raise ValueError("unexpected keyword argument(s) %s" % kw.keys())
    try:
        return _c_functype_cache[(restype, argtypes, flags)]
    except KeyError:
        class CFunctionType(_CFuncPtr):
            _argtypes_ = argtypes
            _restype_ = restype
            _flags_ = flags
        _c_functype_cache[(restype, argtypes, flags)] = CFunctionType
        return CFunctionType

def PYFUNCTYPE(restype, *argtypes):
     class CFunctionType(_CFuncPtr):
         _argtypes_ = argtypes
         _restype_ = restype
         _flags_ = _FUNCFLAG_CDECL | _FUNCFLAG_PYTHONAPI
     return CFunctionType

基本逻辑就是通过打开的动态链接库，根据函数名称，得到一个函数指针，最后通过这个函数指针去调用具体的方法。

_ctypes实现逻辑

  How are functions called, and how are parameters converted to C ?

  1. _ctypes.c::PyCFuncPtr_call receives an argument tuple 'inargs' and a
  keyword dictionary 'kwds'.

  2. After several checks, _build_callargs() is called which returns another
  tuple 'callargs'.  This may be the same tuple as 'inargs', a slice of
  'inargs', or a completely fresh tuple, depending on several things (is it a
  COM method?, are 'paramflags' available?).

  3. _build_callargs also calculates bitarrays containing indexes into
  the callargs tuple, specifying how to build the return value(s) of
  the function.

  4. _ctypes_callproc is then called with the 'callargs' tuple.  _ctypes_callproc first
  allocates two arrays.  The first is an array of 'struct argument' items, the
  second array has 'void *' entries.

  5. If 'converters' are present (converters is a sequence of argtypes'
  from_param methods), for each item in 'callargs' converter is called and the
  result passed to ConvParam.  If 'converters' are not present, each argument
  is directly passed to ConvParm.

  6. For each arg, ConvParam stores the contained C data (or a pointer to it,
  for structures) into the 'struct argument' array.

  7. Finally, a loop fills the 'void *' array so that each item points to the
  data contained in or pointed to by the 'struct argument' array.

  8. The 'void *' argument array is what _call_function_pointer
  expects. _call_function_pointer then has very little to do - only some
  libffi specific stuff, then it calls ffi_call.

  So, there are 4 data structures holding processed arguments:
  - the inargs tuple (in PyCFuncPtr_call)
  - the callargs tuple (in PyCFuncPtr_call)
  - the 'struct arguments' array
  - the 'void *' array
