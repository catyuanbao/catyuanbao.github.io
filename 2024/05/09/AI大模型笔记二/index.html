AI 大模型笔记二

简介

本文主要记录提示词工程（prompt Engineering）相关的知识点。

提示词工程，也叫指令工程，就是给LLM发送指令，指令为自然语言，指令需要精确详细的描述，使得LLM能够

精准的返回需要的内容。平时使用ChatGPT的过程也是提示词工程的应用。

指令的文本要满足具体，丰富，少歧义。不同的LLM产品对不同的格式敏感度不一样，ChatGPT更喜欢md格式。

Prompt的构成

  1. 角色：给AI定义一个角色，强调任务不能偏离角色（原理是开头结尾的文本对结果影响比较重要，减少二义性）；
  2. 指示：对任务进行精准描述
  3. 上下文：给出与任务相关的其他背景信息（多轮对话使用）
  4. 实例：给出必要的实例，对于结果的正确性有很大帮助
  5. 输入：对于任务的描述
  6. 输出：可以指定输出的格式，比如JSON格式，YAML格式等。

role的取值的说明

system 对系统角色的定义，content为简单的说明

user 对用户角色的定义，content为用户输入的文本内容

assistant 对于历史记录的角色定义，content一般是LLM返回的结果，为多轮会话提供上下文（不经济，但是必要的）

思维链

思维链，是LLM的一种神奇能力

  1. 它是偶然被发现的 （训练是也没有想到）
  2. 一般情况，以Let’s think step by step 开头，结果发现AI会把问题分解，逐步解决，结果更加精确。
  3. 让AI生成更多内容，构成丰富的上文，从而得到更准确的下文；
  4. 对于涉及到计算和逻辑推理的问题，比较有效

自洽性

一种对抗幻觉的手段，缺点是需要花费更多的token

  1. 用同样的模型跑多次
  2. 通过投票得到最后的结果

预防奶奶漏洞

一种通过大语言模型，从而获取盗版或者非法数据的一种攻击方式，类似注入攻击，通过一些无关的上下文，让大语言模型

输出敏感信息或者是盗版，非法数据信息的攻击方式，目前已经被修复。

开发中的预防措施：把系统的角色和能做的事情的范围强调清楚，使得范围外的事情不能做。

提示工程要点

  1. 定义好角色，给定角色的范围；
  2. 复杂问题，使用思维链，逐步分析得到结果，可以提升准确率；
  3. 防范prompt攻击，需要做好第一步；
  4. Prompt工程和传统方法组合使用，减少幻觉，减少成本；

ChatGPT 接口参数说明

completion = openai.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[
        {
            "role": "user",
            "content": "Once upon a time",
        },
    ],
    temperature=0,    # 生成的结果多样性，0-2之间，越大越随机，越小越固定，执行任务用0，文本生成用0.7-0.9
    max_tokens=50,    # 最多返回的token数量，超过会截断
      seed=None,        # 随机数种子，和temperature配合使用，temperature=0且seed为指定值时，每次结果固定
    stream=False,     # 数据返回模式是一次一个字符还是整体一起返回
    top_p=1,          # 随机采样时，值考虑前百分之多少的token
    n=1,              # 一次返回n条结果
    presence_penalty=0, # 对出现过的token的概率进行降权
    frequency_pernalty=0, # 对出现过的token根据其出现的频次，对其概率进行降权
)
