简介

大模型，全程”大语言模型”，英文Large Language Model，缩写 LLM

  1. 把大模型当初一个函数，给输入，得到输出；
  2. 任何业务问题，用语言描述，成为大模型输入，得到业务问题结果；
  3. 实际工作中，通常需要将业务拆解为若干子任务，分别解决。理解业务本质，对拆解任务比较有用。

简单说明原理

  1. LLM阅读了大量文本，数据。这个过程称为训练，使用了技术是机器学习；
  2. 把一串token后面跟着的不同token的概率记录下来，记下来的就是参数，也叫权重；
  3. 当我们给它一些token，LLM就能计算出来概率最高的下一个token，过程就是生成，也叫推理；
  4. 用生成的token，再加上上下文，就能继续生成下一个token，从而得到后续的文字；
  5. LLM本身并没有逻辑能力，只是记录了大量的概率，选择出来概率最高的生成结果。

关于第5条LLM自身的回答

是的，您提到的点是正确的。大语言模型（如GPT系列）本身并不具备真正的逻辑推理能力，它们的工作原理主要是基于概率建模和统计学习。这些模型通过学习大量的文本数据，建立了词语之间的关联性，并使用这些关联性来预测下一个可能的词语或句子。因此，它们的输出通常是基于给定上下文中最可能的词语或短语。

虽然这些模型可以表现出某种程度的“推理”能力，比如填补句子的空白或生成相关的文本，但这种“推理”实际上是基于统计模式的预测，而不是真正的逻辑推理。在处理复杂的逻辑问题或推理任务时，大语言模型的性能通常会受到限制。

Token是什么？

  1. 可能是一个单词，也可能是半个，可能是一个汉字，或者半个汉字；
  2. 大模型的训练前，需要把一串文本给原子化，类似分词，把它切割成原子的token；
  3. 国产LLM，一个汉字算1个token，可能比外国产品更节省资金。

这个生成机制的内核叫做 Transformer(变形金刚 https://zh.wikipedia.org/wiki/Transformer%E6%A8%A1%E5%9E%8B)，由Google开发。

用好AI的核心心法

  1. 把AI当人看（ChatGPT已经非常复杂，它的创造者也无法知道它会返回什么，类似你和一个人对话，也不知道会得到什么反馈。）
  2. 用人的思维来理解AI
  3. 用人的思维控制AI
  4. 用人的思维来说服用户正常看待AI的不足与局限。

LLM应用模式

  1. AI 嵌入流程模式（AI embedded）；
  2. AI 辅助模式 (AI Copilot)；
  3. AI 代理模式(AI Agent)；

AI agent 是终极目标，Copilot 是目前追求的状态。

Prompt 工程

类似一问一答的对话式流程，最为简单的实现，调用AI的API，根据返回的结果做直接的使用。

Agent + Function Calling

  1. Agent：AI 主动题反问；
  2. Function Calling ： AI要求执行某个方法。

比如：你问AI，今天是否要带伞，它反问你在哪里（Agent），再调用当地的天气接口（Function Calling），再告诉你答案。

RAG

RAG = Embeddings + 向量数据库

  1. Embeddings：把文本转为数组，方便计算相似度，成为文本向量化
  2. 向量数据库：把向量化的数值保存起来；
  3. 向量搜索：根据输入向量，得到最为相似的向量；
  4. 类似开卷考试，先把问题和答案记录到书上，考试时，把考题和书上的问题比对，找到答案。

Fine-tuning

模型微调，增量训练，教给LLM更多知识。
