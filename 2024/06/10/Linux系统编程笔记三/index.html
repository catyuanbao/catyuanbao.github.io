基本参数

The parameter to malloc(3) is the number of bytes to allocate. 
But what is the size_t data type? Obviously, it's not a C primitive data type; it's a typedef – long
unsigned int on your typical 64-bit platform.

malloc的参数是一个size_t是一个long unsigned int类型，参数不能是负数。

malloc的FAQ

  1. malloc最大可以支持多少内存？

32位系统：理论上，单个进程的地址空间最大为 4GB，但实际可用内存会受到内核和用户空间的划分（通常是 3GB 用户空间，1GB 内核空间）以及其他系统资源的影响。

64位系统：理论上，单个进程的地址空间可以达到 2^64 字节（16EB），但实际可用内存受到物理内存大小和操作系统的限制。

  1. 传一个负数给它会发生什么？

由于 size_t 是无符号的，如果你传递一个负数（例如 -1）给 malloc，这个负数会被转换为一个非常大的正整数。例如，在大多数系统上，size_t 是一个 64 位无符号整数，-1 会被解释为 18446744073709551615 (即 2^64 - 1)。

#include <stdio.h>
#include <stdlib.h>

int main() {
    size_t negative_size = -1;
    void *ptr = malloc(negative_size);

    if (ptr == NULL) {
        printf("Memory allocation failed\n");
    } else {
        printf("Memory allocation succeeded\n");
        free(ptr);
    }

    return 0;
}

运行结果：

  1. malloc是线程安全的嘛？

    在现代的 C 标准库实现中，malloc 及其相关的内存分配函数（如 calloc, realloc, free）通常是线程安全的。

    这意味着它们可以安全地被多个线程同时调用而不会引起数据竞争或未定义行为。这种线程安全性通常通过内部的锁机制来实现。

实例：

#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

#define NUM_THREADS 10

void* thread_func(void* arg) {
    int* data = (int*)malloc(sizeof(int));
    if (data == NULL) {
        printf("Memory allocation failed\n");
        return NULL;
    }
    *data = *((int*)arg);
    printf("Thread %d allocated memory with value %d\n", *data, *data);
    free(data);
    return NULL;
}

int main() {
    pthread_t threads[NUM_THREADS];
    int thread_ids[NUM_THREADS];

    for (int i = 0; i < NUM_THREADS; i++) {
        thread_ids[i] = i;
        if (pthread_create(&threads[i], NULL, thread_func, &thread_ids[i]) != 0) {
            printf("Error creating thread %d\n", i);
            return 1;
        }
    }

    for (int i = 0; i < NUM_THREADS; i++) {
        pthread_join(threads[i], NULL);
    }

    return 0;
}

虽然内部使用锁机制可以保证线程安全，但也可能带来一些性能开销，特别是在高并发情况下。为了提高性能，现代系统中有一些替代方案和优化手段，例如：

  • 线程局部分配器：一些系统提供线程局部的内存分配器，减少了线程之间的锁争用。
  • 高性能内存分配器：例如，tcmalloc (Google Thread-Caching Malloc), jemalloc (Facebook), 以及 Hoard 等第三方内存分配器，设计时考虑了多线程环境下的高性能和可扩展性。

free函数

free函数没有太多可注意的。

  1. The parameter passed to free(3) must be a value returned by one of
    the malloc(3) family APIs (malloc(3), calloc, or realloc[array]).
  2. free has no return value.
  3. Calling free(ptr) does not set ptr to NULL (that would be nice, though). Once freed, do not attempt to use the freed memory.
  4. Do not attempt to free the same memory chunk more than once.

calloc函数

calloc和malloc类似，不过会做初始化，参数也不一样。

使用举例：

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *array;
    size_t num_elements = 10;

    // 使用 calloc 分配内存,包括元素个数，每个元素大小
    array = (int *)calloc(num_elements, sizeof(int));
    if (array == NULL) {
        // 分配失败
        printf("Memory allocation failed\n");
        return 1;
    }

    // 打印数组的元素，所有元素应该初始化为 0
    for (size_t i = 0; i < num_elements; i++) {
        printf("array[%zu] = %d\n", i, array[i]);
    }

    // 使用完毕后释放内存
    free(array);

    return 0;
}

realloc函数

realloc 函数用于调整之前分配的内存块的大小。它可以扩展或缩小已分配的内存块，并在必要时将内存块移动到新的位置。realloc 的原型如下：

void *realloc(void *ptr, size_t size);

ptr：指向之前通过 malloc、calloc 或 realloc 分配的内存块。如果 ptr 为 NULL，realloc 的行为与 malloc 相同。

size：新的内存块大小（以字节为单位）。

使用实例：

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *array;
    size_t initial_size = 5;
    size_t new_size = 10;

    // 使用 malloc 分配初始内存
    array = (int *)malloc(initial_size * sizeof(int));
    if (array == NULL) {
        // 分配失败
        printf("Initial memory allocation failed\n");
        return 1;
    }

    // 初始化前 5 个元素
    for (size_t i = 0; i < initial_size; i++) {
        array[i] = i + 1;
    }

    // 打印初始数组
    printf("Initial array:\n");
    for (size_t i = 0; i < initial_size; i++) {
        printf("array[%zu] = %d\n", i, array[i]);
    }

    // 使用 realloc 扩展内存块
    array = (int *)realloc(array, new_size * sizeof(int));
    if (array == NULL) {
        // 分配失败
        printf("Memory reallocation failed\n");
        return 1;
    }

    // 初始化新分配的内存
    for (size_t i = initial_size; i < new_size; i++) {
        array[i] = i + 1;
    }

    // 打印扩展后的数组
    printf("Reallocated array:\n");
    for (size_t i = 0; i < new_size; i++) {
        printf("array[%zu] = %d\n", i, array[i]);
    }

    // 使用完毕后释放内存
    free(array);

    return 0;
}

reallocarray函数

reallocarray 是一个非标准的 C 函数，最初由 OpenBSD 引入，用于重新分配内存数组。它与 realloc 类似，但提供了一些额外的安全性，特别是在计算新内存块大小时避免整数溢出问题。

使用实例：

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *array;
    size_t initial_size = 5;
    size_t new_size = 10;

    // 使用 calloc 分配初始内存
    array = (int *)calloc(initial_size, sizeof(int));
    if (array == NULL) {
        // 分配失败
        printf("Initial memory allocation failed\n");
        return 1;
    }

    // 初始化前 5 个元素
    for (size_t i = 0; i < initial_size; i++) {
        array[i] = i + 1;
    }

    // 打印初始数组
    printf("Initial array:\n");
    for (size_t i = 0; i < initial_size; i++) {
        printf("array[%zu] = %d\n", i, array[i]);
    }

    // 使用 reallocarray 扩展内存块
    array = (int *)reallocarray(array, new_size, sizeof(int));
    if (array == NULL) {
        // 分配失败
        printf("Memory reallocation failed\n");
        return 1;
    }

    // 初始化新分配的内存
    for (size_t i = initial_size; i < new_size; i++) {
        array[i] = i + 1;
    }

    // 打印扩展后的数组
    printf("Reallocated array:\n");
    for (size_t i = 0; i < new_size; i++) {
        printf("array[%zu] = %d\n", i, array[i]);
    }

    // 使用完毕后释放内存
    free(array);

    return 0;
}
