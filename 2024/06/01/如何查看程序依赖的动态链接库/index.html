背景

学习Cython时想到的，如何查看程序运行时加载的动态库文件，可以通过哪些方法查看？

ldd

NAME
       ldd - print shared object dependencies

SYNOPSIS
       ldd [option]... file...

DESCRIPTION
       ldd  prints the shared objects (shared libraries) required by each program or shared object specified on the command line.  An exam‐
       ple of its use and output (using sed(1) to trim leading white space for readability in this page) is the following:

           $ ldd /bin/ls | sed 's/^ */    /'
               linux-vdso.so.1 (0x00007ffcc3563000)
               libselinux.so.1 => /lib64/libselinux.so.1 (0x00007f87e5459000)
               libcap.so.2 => /lib64/libcap.so.2 (0x00007f87e5254000)
               libc.so.6 => /lib64/libc.so.6 (0x00007f87e4e92000)
               libpcre.so.1 => /lib64/libpcre.so.1 (0x00007f87e4c22000)
               libdl.so.2 => /lib64/libdl.so.2 (0x00007f87e4a1e000)
               /lib64/ld-linux-x86-64.so.2 (0x00005574bf12e000)
               libattr.so.1 => /lib64/libattr.so.1 (0x00007f87e4817000)
               libpthread.so.0 => /lib64/libpthread.so.0 (0x00007f87e45fa000)

ldd可以查看so文件或者可执行文件依赖的共享库。

比如:

$ ldd helloworld.so
    linux-vdso.so.1 (0x00007ffdd6d65000)
    libcrypt.so.2 => /lib64/libcrypt.so.2 (0x00007f44416af000)
    libm.so.6 => /lib64/libm.so.6 (0x00007f44415d4000)
    libc.so.6 => /lib64/libc.so.6 (0x00007f4441200000)
    /lib64/ld-linux-x86-64.so.2 (0x00007f44416fa000)

查看依赖的库。

这个方法只能查看静态的文件，那么如何查看运行的程序，或者即将运行的程序会加载哪些so文件？

strace

strace 工具可以跟踪系统调用和信号。你可以使用 strace 来查看程序在运行时加载的 .so 文件。

这个方法可以查看即将运行的程序会加载什么so文件。

lsof

如果程序以及运行起来了，如何查看已经加载了哪些so文件，是否加载正确，可以使用lsof，查看所有打开的文件：

$ lsof -p 7451 |grep so
