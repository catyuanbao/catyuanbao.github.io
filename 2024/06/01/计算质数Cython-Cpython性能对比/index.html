背景

本文主要通过Cython的官方计算质数的实例，测试Cpython和Cython计算质数的性能对比。

计算方法

Cpython代码实现，cython实现，以及使用cpdef 定义的Cython实现

%%cython -a

import time

def primes_cpython(nb_primes):
    if nb_primes >= 1000000:
       nb_primes = 1000000

    p = []
    n = 2
    while len(p) < nb_primes:
        # Is n prime?
        for i in p:
            if n % i == 0:
                break
        # If no break occurred in the loop
        else:
            p.append(n)
        n += 1
    return p

def primes(int nb_primes):
    # 对使用的变量声明类型
    cdef int n, i, len_p
    cdef int[1000000] p
    
    if nb_primes > 1000000:
        nb_primes = 1000000

    len_p = 0  # The current number of elements in p.
    n = 2
    while len_p < nb_primes:
        # Is n prime?
        for i in p[:len_p]:
            if n % i == 0:
                break
        # If no break occurred in the loop, we have a prime.
        else:
            p[len_p] = n
            len_p += 1
        n += 1
    # Let's copy the result into a Python list:
    result_as_list = [prime for prime in p[:len_p]]
    return result_as_list



cpdef list primes_cpdef(int nb_primes):
    if nb_primes >= 1000000:
        nb_primes = 1000000

    cdef int n = 2
    cdef int i
    cdef int is_prime
    cdef list p = []

    while len(p) < nb_primes:
        is_prime = 1
        for i in p:
            if n % i == 0:
                is_prime = 0
                break
        if is_prime:
            p.append(n)
        n += 1

    return p

max_num = 20000

s = time.time()
primes_cpython(max_num)
print(time.time() - s)

s = time.time()
primes(max_num)
print(time.time() - s)

s = time.time()
primes_cpdef(max_num)
print(time.time() - s)

运行结果：

35.58181715011597
5.372183799743652
9.328117847442627

可以看到CPython的最慢，def 定义的Cython版本最快，cpdef定义的版本中等速度。

这里使用Cython的时候也是有一个编译的过程，运行时top观察会有cc命令在编译。

运行第一次以后，再次运行：

%time primes(20000)

就不需要再编译了。

这里第二个版本使用了预先分配的数组，减少了append方法的调用，会显得和其他版本不一样，使用C++的vector可以实现类似效果：

%%cython -a
# distutils: language=c++

from libcpp.vector cimport vector
import time

def primes_vector(unsigned int nb_primes):

    cdef int n, i
    cdef vector[int] p
    p.reserve(nb_primes)  # allocate memory for 'nb_primes' elements.
    n = 2

    while p.size() < nb_primes:  # size() for vectors is similar to len()
        for i in p:
            if n % i == 0:
                break
        else:
            p.push_back(n)  # push_back is similar to append()
        n += 1

    # If possible, C values and C++ objects are automatically
    # converted to Python objects at need.
    return p  # so here, the vector will be copied into a Python list.

s = time.time()
primes_vector(20000)
print(time.time()-s)

运行时间：

4.017367362976074

import time

    cdef int n, i
    cdef vector[int] p

    return p  # so here, the vector will be copied into a Python list.

    primes_vector(20000)
print(time.time()-s)

运行时间：

11.092583894729614

性能对比表如下：

函数定义方式 运行时间 (秒) 备注
def（Python 版本） 53.48920202255249 纯Python实现，性能最差，因为调用开销和解释器的性能较低。
def（Cython 版本） 7.126289129257202 使用Cython进行编译，虽然是Python函数，但性能有所提升。
cpdef（Cython 版本） 13.231026887893677 可以从Python和Cython调用，性能介于def和cdef之间，从Cython调用时接近cdef，从Python调用时接近def。
cpdef（Cython + C++ vector 版本） 11.092583894729614 使用C++的vector进行优化，在Cython中调用，性能更高。

使用cython的函数第一次需要编译，编译后运行会更快。
