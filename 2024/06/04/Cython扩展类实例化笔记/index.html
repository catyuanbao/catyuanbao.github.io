背景

本文主要记录Cython扩展类实例化相关的知识，使用cdef不光可以修饰函数，也可以修饰类。

这种类就称为扩展类，扩展类的实例化，可以支持手动的内存分配回收机制，性能更高。

什么是Cython扩展类

在 Cython 中，使用 cdef 定义的类被称为 “extension types” 或 “cdef classes”。

这些类与普通的 Python 类有一些显著的不同，主要在性能优化和内存管理方面。cdef 类提供了对底层 C 层面的更直接控制，因此它们通常比普通 Python 类更高效。

特点

高效性能：由于 cdef 类直接编译为 C 代码，所以它们的属性访问、方法调用和实例创建都比普通 Python 类更快。

更严格的类型检查：cdef 类的属性类型是静态的，在编译时确定，这减少了运行时的类型检查开销。

无法继承自纯 Python 类：cdef 类不能直接继承自普通的 Python 类，反之亦然。

限制性：cdef 类的实例不能直接与 Python 的内置类型进行交互，除非通过显式的接口转换。

简单实例

cdef class MyCythonClass:
    cdef int attribute1
    cdef double attribute2

    def __cinit__(self, int attribute1, double attribute2):
        self.attribute1 = attribute1
        self.attribute2 = attribute2

    def set_values(self, int attribute1, double attribute2):
        self.attribute1 = attribute1
        self.attribute2 = attribute2

    def get_sum(self) -> double:
        return self.attribute1 + self.attribute2

这里有一个__cinit__方法。

__cinit__是负责执行C级别的类实例属性(指针类型)的内存分配和初始化，并且在外部代码的执行结束前，Cython编译器会默认隐含调用扩展类实例的 __dealloc__方法，这些行为和C++编译器是一致。

__cinit__等同C++类中默认构造函数

__dealloc__ 等同于C++类的析构函数

一个完整点的实例：

%%cython -a

from libc.stdlib cimport malloc,free

cdef class Fruit(object):
    '''Fruit Type'''
    
    cdef readonly str name
    cdef public double qty
    cdef readonly double price
    cdef double *weight
    
    def __cinit__(self,nm,qt,pc):
        print("__cinit__ method executed")
        self.name=nm
        self.qty=qt
        self.price=pc
        self.weight=<double*>malloc(sizeof(double))
        self.weight[0]=23.33
        
        print("weight: ",self.weight[0])
        
    def __init__(self,nm,qt,pc):
        print("__init__ method executed")

    def amount(self):
        return  self.qty*self.price
    
    def __dealloc__(self):
        print("__dealloc__method executed")
        
        if self.weight!=NULL:
            free(self.weight)


apple = Fruit('apple', 1, 1)
print(apple)
del apple

这里可以看到，使用del删除了对象也会自动调用__dealloc__方法。

使用__cinit__注意点

  • __cinit__可能会带来额外的开销
  • __cinit__的参数声明和__init__必须一致，因为它们会被同时调用，因此__cinit__的参数会留下*kargs,**kwargs
  • __cinit__中涉及到C指针类型的类属性的内存分配，必须在类定中显式定义__dealloc__，因为需要通过__dealloc __对C指针类型的实例属性进行内存释放，类似malloc和free成对出现一样。
  • __cinit__和__init__只能使用def关键字申明，不能使用cdef
  • Cython扩展类在构造时会保证__cinit__只能被调用一次，并且在__init__或其他Python级别的构造函数（例如，类方法构造函数）之前被调用。
  • cdef定义的扩展类中，不能包含__new__方法重写，因为__cinit__中包含了__new__的功能。

显式调用快速实例化对象

对于Cython扩展类来说，我们使用显式调用Fruit.__new__会比常规Fruit(…)语句执行实例化会少一个步骤，因为显式调__会默认只执行Fruit.__cinit__构造函数，不执行__init__方法，因此，会更快。

因此，使用__new__的方法调用，比直接调用性能更高。

测试用例：

%%cython -a

from libc.stdlib cimport malloc,free

cdef class Fruit(object):
    '''Fruit Type'''
    
    cdef readonly str name
    cdef public double qty
    cdef readonly double price
    cdef double *weight
    
    def __cinit__(self,nm,qt,pc):
        # print("__cinit__ method executed")
        self.name=nm
        self.qty=qt
        self.price=pc
        self.weight=<double*>malloc(sizeof(double))
        self.weight[0]=23.33
        
        # print("weight: ",self.weight[0])
        
    def __init__(self,nm,qt,pc):
        pass
        # # print("__init__ method executed")

    def amount(self):
        return  self.qty*self.price
    
    def __dealloc__(self):
        # print("__dealloc__method executed")
        
        if self.weight!=NULL:
            free(self.weight)


apple = Fruit('apple', 1, 1)
print(apple)

%timeit -n 100000 Fruit('apple', 1, 1)  # 710 ns ± 156 ns per loop (mean ± std. dev. of 7 runs, 100,000 loops each)
%timeit -n 100000 Fruit.__new__(Fruit, 'apple', 1, 1) # 1.01 µs ± 112 ns per loop (mean ± std. dev. of 7 runs, 100,000 loops each)

会比隐式调用__new__更快。

使用@cython.freelist()缓存对象

在 Cython 中，freelist 是一种优化技术，用于减少对象分配和释放的开销。这种技术通过维护一个空闲列表（freelist），在对象被销毁时将其加入空闲列表，而不是立即释放内存。当需要创建新的对象时，如果空闲列表中有可用对象，则直接复用这些对象，而不进行新的内存分配。相当于数据库的软删除，或者缓存。

使用实例：

cimport cython


@cython.freelist(8)
cdef class Penguin:
    cdef object food
    def __cinit__(self, food):
        self.food = food

penguin = Penguin('fish 1')
penguin = None
penguin = Penguin('fish 2')  # does not need to allocate memory!

那么如何从底层验证freelist生效呢？

这里需要hook到malloc:

hook_malloc_free.c:

// hook_malloc_free.c
#define _GNU_SOURCE
#include <stdio.h>
#include <dlfcn.h>

static int malloc_count = 0;
static int free_count = 0;

void* malloc(size_t size) {
    void* (*real_malloc)(size_t) = dlsym(RTLD_NEXT, "malloc");
    malloc_count++;
    return real_malloc(size);
}

void free(void* ptr) {
    void (*real_free)(void*) = dlsym(RTLD_NEXT, "free");
    free_count++;
    real_free(ptr);
}

void print_memory_allocations() {
    printf("Malloc count: %d\n", malloc_count);
    printf("Free count: %d\n", free_count);
}

编译：

gcc -shared -fPIC hook_malloc_free.c -o libhook.so -ldl

hook.pyx:

cimport cython

@cython.freelist(3)
cdef class MyClass:
    cdef object name
    def __cinit__(self, name):
        self.name = name
    def __init__(self, name):
        self.name = name

def count_allocations():
    cdef extern void print_memory_allocations()
    print_memory_allocations()

def test_freelist_reuse():
    count_allocations()
    obj1 = MyClass('1')
    count_allocations()
    obj2 = MyClass('2')
    count_allocations()
    obj3 = MyClass('3')
    count_allocations()
    obj4 = MyClass('4')
    count_allocations()
    obj5 = MyClass('5')
    count_allocations()
    obj6 = MyClass('6')
    count_allocations()
    obj7 = MyClass('7')
    count_allocations()
    obj8 = MyClass('8')
    count_allocations()
    obj9 = MyClass('9')
    count_allocations()

test_freelist_reuse()
