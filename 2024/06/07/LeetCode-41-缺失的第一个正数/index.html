背景

本文主要记录解决LeetCode 41 缺失的第一个正数过程。

这个题目，老实话，没看懂最后一个实例的意思：

示例 3：

输入：nums = [7,8,9,11,12]
输出：1
解释：最小的正数 1 没有出现。

看了题解，意思是从[1,最大值]，中找到，没有排序的数字中最小的一个没出现的正数的值。

先按照使用额外空间的方法实现以下，确保理解没有问题：

Python语言实现，使用额外的空间，这个实现的复杂度（时间+空间）都不满足要求的。题目要求O(n)，没有额外空间，

这里是O(最大值)，可能超过N，具体看最大值是多少，空间使用了额外的hash表，不满足要求。

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        max_num = max(map(abs, nums)) + 1
        cache = {}
        for i in nums:
            cache[i] = i

        for i in range(1, max_num +1):
            if cache.get(i) is None:
                return i

这里不能用额外的空间，那就必须对原始数据做处理。

这里官方给了2种方法，一种是置换法：

类似于排序，把第x-1个元素置为x，如果x在1-n之间。
[3, 4, -1, 1] 改为 [1, -1, 3, 4]，缺失的就是2。

算法如下：

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)
        for i in range(n):
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                nums[nums[i] - 1], nums[i] = nums[i], nums[nums[i] - 1]
        # 没有在正确的位置
        for i in range(n):
            if nums[i] != i + 1:
                return i + 1
        # 最后返回n+1
        return n + 1

这里 第一个for循环下面为什么不用if，而是要while, while 而不是 if 的原因在于，有些数字在一次交换之后，新的数字也需要被放置到它的正确位置。

如果仅使用 if，一次交换之后并不能保证所有数字都在正确位置上，需要多次遍历数组，这样效率会很低。
