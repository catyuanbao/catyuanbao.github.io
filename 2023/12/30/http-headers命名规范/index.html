http headers命名规范

HTTP头字段（headers）的key（字段名）在RFC 2616（HTTP/1.1规范）中有一些基本的规则：

  1. 大小写不敏感： HTTP头字段的名字是大小写不敏感的，也就是说，”Content-Type”和”content-type”是等效的。
  2. 不允许空格： 头字段名不允许包含空格，且不能以空格开头。
  3. 只允许ASCII字符： HTTP头字段名只能包含ASCII字符。
  4. 避免使用下划线： RFC 2616建议避免在头字段名中使用下划线（_），建议使用连字符（-）代替。例如，使用”Content-Type”而不是”Content_Type”。
  5. 不允许出现重复的字段名： 除非特定的头字段定义允许重复，否则头字段名不能重复。例如，Accept头字段可以包含多个值，但大多数头字段不允许这样做。

不同语言的不同框架对于基本规范的实现不用，这里使用Go语言的Gin框架和Python语言的Flask框架测试。

Gin框架服务端代码：

package main

import (
        "fmt"
        "github.com/gin-gonic/gin"
)

func main() {
        router := gin.Default()

        router.GET("/", func(c *gin.Context) {
                // 获取所有的请求头信息
                headers := c.Request.Header

                // 遍历并打印每个头信息
                for key, value := range headers {
                        fmt.Printf("%s: %s\n", key, value)
                }

                c.String(200, "Check the console for printed headers")
        })

        router.Run(":8080")
}

测试不允许空格

import requests

headers = {
    'a_id': 'a id',
    'hello-world': 'hello',

    'a': '123',
    'lorem sss': 'sss'

resp = requests.get('http://localhost:8080', headers=headers)

使用Flask框架测试：

Flask服务端代码：

from flask import Flask, request

app = Flask(__name__)

@app.route('/')
def print_headers():
    # 获取所有的请求头信息
    headers = request.headers

    # 遍历并打印每个头信息
    for key, value in headers.items():
        print(f"{key}: {value}")

    return "Check the console for printed headers"

if __name__ == '__main__':
    app.run(debug=True)

测试使用下划线

import requests

headers = {
    'a_id': 'a id',
    'hello-world': 'hello',
    'hello_world': 'world',
    'a': '123',
}

resp = requests.get('http://localhost:5000', headers=headers)
print(resp.text)

服务端打印的headers:

Host: localhost:5000
User-Agent: python-requests/2.28.2
Accept-Encoding: gzip, deflate, br
Accept: */*
Connection: keep-alive
A-Id: a id
Hello-World: hello,world
A: 123

这里会直接把带有下划线的字段转为中划线，然后把字段的值给合并了。

测试带有空格的字段

import requests

headers = {
    'a_id': 'a id',
    'hello-world': 'hello',
    'hello_world': 'world',
    'a': '123',
    'with space': 'none'
}

resp = requests.get('http://localhost:5000', headers=headers)
print(resp.text)

请求是成功的

服务端打印的headers:

Host: localhost:5000
User-Agent: python-requests/2.28.2
Accept-Encoding: gzip, deflate, br
Accept: */*
Connection: keep-alive
A-Id: a id
Hello-World: hello,world
A: 123

会把带有空格的字段直接忽略。

这里可以看出，Flask对http headers的标准规则遵守的不是很严格，Gin框架实现的比较严谨，不同的框架可能实现都不一样，在实现功能的时候遵守基本的规则，不容易出错。
